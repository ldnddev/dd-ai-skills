# WCAG 2.2 Checklist
<!-- Updated: 2026-04-19 -->

## Perceivable - Information and user interface components must be presentable to users in ways they can perceive.

### 1.1 Text Alternatives (Level A)
- [ ] 1.1.1 Non-text Content: All non-text content has a text alternative
- [ ] Images have descriptive alt text
- [ ] Decorative images have empty alt text (alt="")
- [ ] Complex images have long descriptions
- [ ] Form images have descriptive alt text
- [ ] CAPTCHAs have audio alternatives

### 1.2 Time-based Media (Level A/AA/AAA)
- [ ] 1.2.1 Audio-only and Video-only (Prerecorded): Alternatives provided
- [ ] 1.2.2 Captions (Prerecorded): Synchronized captions for video
- [ ] 1.2.3 Audio Description or Media Alternative: Audio descriptions
- [ ] 1.2.4 Captions (Live): Real-time captions for live audio
- [ ] 1.2.5 Audio Description (Prerecorded): Extended audio descriptions
- [ ] 1.2.6 Sign Language (Prerecorded): Sign language interpretation
- [ ] 1.2.7 Extended Audio Description (Prerecorded): Extended descriptions
- [ ] 1.2.8 Media Alternative (Prerecorded): Full text alternative
- [ ] 1.2.9 Audio-only (Live): Text alternative for live audio

### 1.3 Adaptable (Level A)
- [ ] 1.3.1 Info and Relationships: Semantic HTML used correctly
- [ ] 1.3.2 Meaningful Sequence: Logical reading order
- [ ] 1.3.3 Sensory Characteristics: Instructions not reliant on sensory characteristics
- [ ] 1.3.4 Orientation: Content not restricted to portrait or landscape
- [ ] 1.3.5 Identify Input Purpose: Input purposes programmatically determinable
- [ ] 1.3.6 Identify Purpose: Purpose of UI components programmatically determinable

### 1.4 Distinguishable (Level A/AA/AAA)
- [ ] 1.4.1 Use of Color: Color not used as only visual means of conveying information
- [ ] 1.4.2 Audio Control: Mechanism to pause/stop/control audio
- [ ] 1.4.3 Contrast (Minimum): Text contrast ratio of at least 4.5:1
- [ ] 1.4.4 Resize Text: Text can be resized up to 200% without loss of content
- [ ] 1.4.5 Images of Text: Text not presented as images unless essential
- [ ] 1.4.6 Contrast (Enhanced): Text contrast ratio of at least 7:1
- [ ] 1.4.7 Low or No Background Audio: Background sounds can be turned off
- [ ] 1.4.8 Visual Presentation: Text spacing and line height adjustable
- [ ] 1.4.9 Images of Text (No Exception): Text never presented as images
- [ ] 1.4.10 Reflow: Content reflows without horizontal scrolling at 400% zoom
- [ ] 1.4.11 Non-text Contrast: UI components have 3:1 contrast ratio
- [ ] 1.4.12 Text Spacing: Text spacing can be adjusted
- [ ] 1.4.13 Content on Hover or Focus: Hover/focus content is dismissible, hoverable, persistent

## Operable - User interface components and navigation must be operable.

### 2.1 Keyboard Accessible (Level A)
- [ ] 2.1.1 Keyboard: All functionality operable via keyboard
- [ ] 2.1.2 No Keyboard Trap: No keyboard traps
- [ ] 2.1.3 Keyboard (No Exception): All functionality operable without specific timings
- [ ] 2.1.4 Character Key Shortcuts: Shortcuts can be turned off or remapped

### 2.2 Enough Time (Level A/AA/AAA)
- [ ] 2.2.1 Timing Adjustable: Time limits can be turned off, adjusted, or extended
- [ ] 2.2.2 Pause, Stop, Hide: Moving/blinking content can be paused, stopped, or hidden
- [ ] 2.2.3 No Timing: No timing is essential to the activity
- [ ] 2.2.4 Interruptions: Interruptions can be postponed or suppressed
- [ ] 2.2.5 Re-authenticating: Session does not expire during data entry
- [ ] 2.2.6 Timeouts: Users warned of impending timeout

### 2.3 Seizures and Physical Reactions (Level A)
- [ ] 2.3.1 Three Flashes or Below Threshold: No content flashes more than 3 times per second
- [ ] 2.3.2 Three Flashes: No content flashes more than 3 times per second
- [ ] 2.3.3 Animation from Interactions: Motion animation can be disabled

### 2.4 Navigable (Level A/AA)
- [ ] 2.4.1 Bypass Blocks: Mechanism to skip repeated content
- [ ] 2.4.2 Page Titled: Pages have descriptive titles
- [ ] 2.4.3 Focus Order: Logical tab order
- [ ] 2.4.4 Link Purpose (In Context): Link purpose clear from link text alone
- [ ] 2.4.5 Multiple Ways: Multiple ways to locate pages
- [ ] 2.4.6 Headings and Labels: Descriptive headings and labels
- [ ] 2.4.7 Focus Visible: Keyboard focus indicator visible
- [ ] 2.4.8 Location: User's location within set of pages is clear
- [ ] 2.4.9 Link Purpose (Link Only): Link purpose clear from link text
- [ ] 2.4.10 Section Headings: Content organized with headings
- [ ] 2.4.11 Focus Not Obscured (Minimum): Focus indicator not completely hidden
- [ ] 2.4.12 Focus Not Obscured (Enhanced): Focus indicator not hidden at all
- [ ] 2.4.13 Focus Appearance: Focus indicator meets size and contrast requirements

### 2.5 Input Modalities (Level A)
- [ ] 2.5.1 Pointer Gestures: All functionality operable with simple pointer gestures
- [ ] 2.5.2 Pointer Cancellation: Pointer down events can be cancelled
- [ ] 2.5.3 Label in Name: UI components have accessible names matching visible labels
- [ ] 2.5.4 Motion Actuation: Functionality can be operated without device motion
- [ ] 2.5.5 Target Size: Touch targets at least 44×44 CSS pixels
- [ ] 2.5.6 Concurrent Input Mechanisms: Content does not restrict input method
- [ ] 2.5.7 Dragging Movements: Dragging can be done with single pointer
- [ ] 2.5.8 Target Size (Enhanced): Touch targets at least 44×44 CSS pixels with exceptions

## Understandable - Information and operation of user interface must be understandable.

### 3.1 Readable (Level A)
- [ ] 3.1.1 Language of Page: Default human language identified
- [ ] 3.1.2 Language of Parts: Changes in human language identified
- [ ] 3.1.3 Unusual Words: Mechanism for identifying definitions
- [ ] 3.1.4 Abbreviations: Mechanism for identifying expanded form
- [ ] 3.1.5 Reading Level: Text written at lower secondary education level
- [ ] 3.1.6 Pronunciation: Mechanism for identifying pronunciation

### 3.2 Predictable (Level A)
- [ ] 3.2.1 On Focus: Components do not change context on receiving focus
- [ ] 3.2.2 On Input: Changing settings does not automatically change context
- [ ] 3.2.3 Consistent Navigation: Navigation consistent across pages
- [ ] 3.2.4 Consistent Identification: Components with same functionality identified consistently
- [ ] 3.2.5 Change on Request: Context changes only at user request

### 3.3 Input Assistance (Level A/AA)
- [ ] 3.3.1 Error Identification: Errors identified and described
- [ ] 3.3.2 Labels or Instructions: Labels or instructions provided
- [ ] 3.3.3 Error Suggestion: Suggestions provided for correcting errors
- [ ] 3.3.4 Error Prevention (Legal, Financial, Data): Reversible submissions, checked, confirmed
- [ ] 3.3.5 Help: Context-sensitive help available
- [ ] 3.3.6 Error Prevention (All): All user input errors can be corrected

## Robust - Content must be robust enough to be interpreted reliably by a wide variety of user agents, including assistive technologies.

### 4.1 Compatible (Level A)
- [ ] 4.1.1 Parsing: Markup validated and well-formed
- [ ] 4.1.2 Name, Role, Value: UI components have accessible names, roles, values
- [ ] 4.1.3 Status Messages: Status messages programmatically determinable
- [ ] 4.1.4 Notifications: Notifications programmatically determinable

## Compliance Levels

### Level A (Minimum)
- Essential for basic accessibility
- 25 success criteria
- Required for legal compliance in most jurisdictions

### Level AA (Standard)
- Addresses major barriers for disabled users
- 13 additional success criteria (38 total A+AA)
- Required for ADA, Section 508, AODA compliance

### Level AAA (Enhanced)
- Highest level of accessibility
- 23 additional success criteria (61 total)
- Not required for most compliance, but recommended for public services

## Mobile-Specific Considerations

### Touch Targets
- [ ] Minimum 44×44 CSS pixels for touch targets
- [ ] Adequate spacing between touch targets
- [ ] Touch targets not overlapping

### Gestures
- [ ] Complex gestures have simple alternatives
- [ ] Swipe gestures can be performed with single pointer
- [ ] Pinch/zoom gestures not required for core functionality

### Viewport
- [ ] Content reflows at 400% zoom
- [ ] No horizontal scrolling at 320px width
- [ ] Text remains readable when zoomed

## Testing Tools Integration

### Automated Testing (axe-core)
- Tests ~30-40% of WCAG 2.2 criteria
- Focuses on programmatically detectable issues
- Use with `--level AA` for standard compliance

### Manual Testing Requirements
- Keyboard navigation testing
- Screen reader testing (NVDA, VoiceOver, JAWS)
- Color contrast verification
- Focus management testing
- Mobile device testing

## Legal Compliance Mapping

### ADA (Americans with Disabilities Act)
- Requires WCAG 2.0/2.1 Level AA
- Applies to public accommodations and commercial facilities

### Section 508 (U.S. Federal)
- Requires WCAG 2.0 Level AA
- Applies to federal agencies and contractors

### AODA (Accessibility for Ontarians with Disabilities Act)
- Requires WCAG 2.0 Level AA
- Applies to organizations in Ontario, Canada

### EN 301 549 (European)
- Requires WCAG 2.1 Level AA
- Applies to public sector bodies in EU
